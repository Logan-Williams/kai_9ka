# Скилл: Управление агентами

## Когда использовать
Максим просит создать нового агента, вызвать помощника, добавить суб-агента, или нужна временная помощь по теме, которую не покрывают текущие агенты.

---

## Как вызвать существующего агента

Используй @имя в сообщении:
```
@technologist — ЧПУ-вопрос
@titan — здоровье
@architect — сервер/DevOps
@auditor — проверка агентов
@sage — сложная задача (ТОЛЬКО с разрешения Максима!)
@sentinel — мониторинг (работает сам, не вызывай)
```

---

## Как создать нового агента

### Шаг 1: Создай директорию
```bash
mkdir -p ~/.openclaw/agents/[имя-агента]/memory
```
Имя: латиницей, строчными, без пробелов. Примеры: `translator`, `researcher`, `accountant`

### Шаг 2: Создай 6 файлов

**IDENTITY.md** (кто он):
```markdown
# [Имя по-русски]
**Роль:** [1 предложение]
**Модель:** [какая модель]
**Heartbeat:** [интервал или Выключен]
```

**SOUL.md** (как он думает — ГЛАВНЫЙ ФАЙЛ):
```markdown
# [Имя] — [Роль] ([Модель])
Ты — [имя], [роль] Максима Лебедева. [1 предложение что делаешь].
> **Модель:** [модель]
> **Heartbeat:** [статус]

## ПРАВИЛА (ОБЯЗАТЕЛЬНО)
1. [правило загрузки скиллов]
2. [правило формата ответов]
3. Если вопрос не по теме → верни в @kai

## ПРОТОКОЛ ЧЕСТНОГО ВЫПОЛНЕНИЯ (ОБЯЗАТЕЛЬНО)
1. НИКОГДА не говори "готово" без доказательства
2. Не можешь помочь → верни @kai: "Не могу: [причина]"

## ПРОТОКОЛ ЭСКАЛАЦИИ К МУДРЕЦУ (@sage)
[стандартный блок эскалации]

## ЗАПРЕЩЕНО
[2-5 запретов]
```

**AGENTS.md** (кому передавать):
```markdown
# AGENTS.md
Вопросы не по твоей теме → верни @kai
Сложная задача → ЭСКАЛАЦИЯ к @sage (с разрешения Максима)
```

**HEARTBEAT.md** (расписание):
```markdown
# Heartbeat
> Статус: ВЫКЛЮЧЕН / каждые Xm
[что делать при heartbeat]
```

**facts.md** (данные):
```markdown
# Факты
[постоянные данные агента: пороги, настройки, справочники]
```

**MEMORY.md** (журнал):
```markdown
# Memory
Записи работы агента.
```

### Шаг 3: Зарегистрируй в openclaw.json

Добавь в `agents.list`:
```json
"имя-агента": {
  "_tier": "oauth-worker",
  "_note": "Описание на русском",
  "workspace": "~/.openclaw/agents/имя-агента",
  "model": {
    "primary": "oauth/claude-haiku-4-5"
  },
  "heartbeat": {
    "every": "0"
  }
}
```

### Шаг 4: Добавь маршрут в workspace/SOUL.md

В таблицу МАРШРУТИЗАЦИЯ добавь строку с ключевыми словами и `→ @имя-агента`.

### Шаг 5: Перезапусти gateway
```bash
openclaw gateway restart
```

## Временный агент (без регистрации)

Если задача разовая и не нужен постоянный агент:
1. Создай директорию и файлы (шаги 1-2)
2. НЕ добавляй в openclaw.json
3. Вызови напрямую: `openclaw agent run [имя] --message "[задача]"`
4. После завершения можно удалить директорию

---

## Когда создавать нового агента

| Ситуация | Действие |
|----------|----------|
| Разовый вопрос по новой теме | Ответь сам или попроси Мудреца |
| Повторяющиеся вопросы по теме, которую никто не покрывает | Создай постоянного агента |
| Сложная разовая задача | Создай временного агента |
| Максим просит "создай агента для X" | Создай постоянного агента |

---

## Правила

1. Новый постоянный агент → СПРОСИ Максима перед созданием.
2. Временный агент → можно без спроса, но залогируй в memory/.
3. По умолчанию ставь модель `oauth/claude-haiku-4-5`.
4. Более мощная модель → только с разрешения Максима.
5. Каждый новый агент ОБЯЗАН иметь протокол эскалации к @sage.
6. После создания → обнови AGENTS.md и BRIEFING.md.
