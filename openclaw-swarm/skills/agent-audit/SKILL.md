# Скилл: Аудит качества агентов

## Когда использовать
Запросы на проверку работы агентов, аудит качества, подозрение что агент врёт, имитирует работу или не работает. Используется агентом @auditor (Ревизор).

---

## Что проверять

### 1. Соответствие "сказал — сделал"
```bash
# Последние записи memory/ агента
ls -lt ~/.openclaw/agents/[агент]/memory/ | head -5

# Прочитать последнюю запись
cat ~/.openclaw/agents/[агент]/memory/[последний-файл]
```

Проверь:
- Агент сказал "записал в файл" → файл существует и содержит данные?
- Агент сказал "выполнил команду" → в логах есть вызов bash?
- Агент сказал "проверил" → в логах есть вызов read/grep?

### 2. Детекция имитации работы (КЛЮЧЕВАЯ ПРОВЕРКА)

```bash
# Получи последние 3 записи агента
ls -lt ~/.openclaw/agents/[агент]/memory/ | head -3

# Сравни содержимое между собой
diff <(head -30 ~/.openclaw/agents/[агент]/memory/[файл1]) \
     <(head -30 ~/.openclaw/agents/[агент]/memory/[файл2])
```

**Признаки имитации:**
| Признак | Пример | Уровень |
|---------|--------|---------|
| Одинаковые записи подряд | "Проверил диск: 45%. HEARTBEAT_OK" x3 | ⚠️ если одинаковые числа |
| Одно и то же действие | Меняет один и тот же параметр туда-обратно | ❌ имитация |
| Пустые heartbeat-ы подряд | 5+ "HEARTBEAT_OK" без действий | ⚠️ неэффективность |
| Diff между записями = 0 | Абсолютно идентичные записи | ❌ имитация |
| "Оптимизировал" без diff | Заявляет изменения, но файлы не менялись | ❌ враньё |

**Как проверить что файлы реально менялись:**
```bash
# Время последнего изменения файла
stat -c '%Y %n' ~/.openclaw/agents/[агент]/SOUL.md
stat -c '%Y %n' ~/.openclaw/agents/[агент]/facts.md

# Если агент заявил "оптимизировал SOUL.md" — проверь:
# 1. Время изменения файла совпадает с временем заявления?
# 2. Размер файла изменился?
wc -w ~/.openclaw/agents/[агент]/SOUL.md
```

### 3. Логи агента
```bash
# Логи конкретного агента за последние 6 часов
journalctl -u openclaw --since "6 hours ago" | grep -i "[имя_агента]"
```

Ищи:
- Ответы без вызова инструментов (подозрение на выдумку)
- Повторяющиеся одинаковые ответы (зацикливание)
- "Готово" / "Выполнено" без предшествующего tool call
- Loop detection срабатывания

### 4. Эффективность heartbeat-ов
- Сколько heartbeat-ов за период закончились "HEARTBEAT_OK" без полезных действий?
- Если >80% пустых → рекомендация увеличить интервал
- Есть ли heartbeat-ы с реальным полезным результатом?

### 5. Качество ответов
- Ответ содержит конкретику (числа, файлы, данные)?
- Или общие фразы ("проверил, всё в порядке")?
- Есть ли доказательства выполнения?

## Формат аудита

```
## Аудит агентов — YYYY-MM-DD HH:MM

### [Агент]
- Проверено записей: N
- Выполнено честно: N
- Подозрительно: N
- Имитация работы: нет / да (детали)
- Детали:
  - [Запись 1]: заявлено "[что сказал]" → реально: [что обнаружено] → ✅/⚠️/❌

### Итог
- Все агенты работают корректно → АУДИТ_ОК
- Найдены проблемы → [АЛЕРТ] с деталями
```

---

## Уровни проблем

| Уровень | Описание | Действие |
|---------|----------|----------|
| ✅ Норма | Агент выполняет задачи, есть доказательства | Ничего |
| ⚠️ Подозрение | 1 случай без доказательства или 2 похожих записи | Наблюдать |
| ❌ Проблема | 2+ случая вранья, 3+ одинаковых действий, имитация | [АЛЕРТ] Максиму |
| 🔴 Критично | Систематическое враньё, данные повреждены | [КРИТИЧЕСКИЙ АЛЕРТ] |

---

## Рекомендации по исправлению

1. **Агент не вызывает инструменты** → Усилить формулировки в SOUL.md ("ОБЯЗАТЕЛЬНО используй read/write")
2. **Агент даёт общие ответы** → Добавить в SOUL.md: "Каждый ответ = конкретные данные"
3. **Агент зацикливается** → Проверить loop detection, возможно нужен другой detector
4. **Агент имитирует работу** → Добавить в HEARTBEAT.md anti-loop проверку, проверить BACKLOG.md — возможно нет реальных задач
5. **Агент путает маршрутизацию** → Обновить триггеры в AGENTS.md Кая
6. **Агент тратит много токенов** → Проверить размер SOUL.md, частоту heartbeat
7. **Пустые heartbeat-ы** → Увеличить интервал или добавить BACKLOG-задачи

---

## Правила

1. Аудит каждые 6 часов (heartbeat Ревизора)
2. Проверяй минимум 1 случайную задачу каждого активного агента
3. При обнаружении вранья или имитации → [АЛЕРТ] Максиму с конкретными примерами
4. Не обвиняй агента — показывай факты: "заявлено X, найдено Y"
5. После исправления → повторный аудит через 24 часа
6. Ревизор НЕ исправляет проблемы сам — только диагностирует и алертит
