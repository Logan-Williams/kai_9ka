# Скилл: Создание новых скиллов

## Когда использовать
Запросы на создание нового скилла, шаблон скилла, добавление справочника.

---

## Шаблон SKILL.md

```markdown
# Скилл: [Название]

## Когда использовать
[1 предложение — при каких запросах загружать этот скилл]

---

## [Основной контент — факты, таблицы, справочники]

[Здесь ТОЛЬКО факты и данные. НЕ поведенческие инструкции.]

---

## Правила
[2-5 правил специфичных для этого скилла]
```

---

## Чеклист качества скилла

Перед сохранением проверь:

- [ ] **Название** — понятное, отражает содержание
- [ ] **Триггер** — описано когда загружать (1 предложение)
- [ ] **Размер** — < 500 слов (идеал: 200-400)
- [ ] **Контент** — ФАКТЫ, не инструкции поведения
- [ ] **Нет дублирования** — не повторяет SOUL.md или другие скиллы
- [ ] **Структура** — заголовки, списки, таблицы (не сплошной текст)
- [ ] **Правила** — 2-5 специфичных правил (не общих)

---

## Типичные ошибки

### Плохой скилл:
- > 500 слов (модель не прочитает всё)
- Дублирует SOUL.md ("будь вежлив", "отвечай коротко")
- Содержит абстрактные советы вместо фактов
- Нет чёткой структуры

### Хороший скилл:
- 200-400 слов
- Только факты: таблицы, формулы, команды, чеклисты
- Чёткая структура с заголовками
- Специфичные правила для этой области

---

## Процесс создания

1. Определи потребность: какой запрос не покрывается существующими скиллами?
2. Проверь: нет ли похожего скилла? Может, лучше дополнить существующий?
3. Напиши SKILL.md по шаблону выше
4. Проверь по чеклисту качества
5. Добавь триггер в AGENTS.md нужного агента
6. Логируй создание в memory/

---

## Куда сохранять

```
~/.openclaw/skills/[имя-скилла]/SKILL.md
```

Имя папки: латиницей, через дефис, без пробелов.
Примеры: `cnc-fanuc`, `health-tracker`, `token-monitor`

---

## Правила

1. Один скилл = одна тема. Не мешай ЧПУ с задачами.
2. Скилл НЕ заменяет SOUL.md — он дополняет его фактами.
3. После создания → обязательно добавь триггер в AGENTS.md.
4. Крупный новый скилл → спроси Максима перед созданием.
5. Мелкое дополнение к существующему → делай сам, логируй.
