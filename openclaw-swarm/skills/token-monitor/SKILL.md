# Скилл: Мониторинг токенов

## Когда использовать
Запросы о расходе токенов, бюджете, стоимости, аномалиях в расходе.

---

## Команды

```bash
# Общий статус и расход
openclaw status

# Логи за последний час (ищем обращения к API)
journalctl -u openclaw -n 50 --since "1 hour ago" | grep -i "token\|cost\|usage"

# Логи за сутки
journalctl -u openclaw --since "24 hours ago" | grep -i "token\|cost\|usage"
```

## Бюджет

| Уровень | Расход/день | Действие |
|---------|-------------|----------|
| Нормальный | $1-3 | HEARTBEAT_OK |
| Повышенный | $3-5 | Наблюдать, логировать |
| Алерт | $5-8 | [АЛЕРТ] Максиму + анализ причины |
| Критический | >$8 | [КРИТИЧЕСКИЙ АЛЕРТ] + предложить меры |

## Типичные причины перерасхода

1. **Зацикленный агент** — повторяет одно действие
   - Признак: одинаковые логи каждые N минут
   - Решение: loop detection должен поймать, проверить логи

2. **Слишком частый heartbeat** — агент вызывается без дела
   - Признак: heartbeat без полезных действий в логах
   - Решение: увеличить интервал или отключить

3. **Длинный SOUL.md** — модель перечитывает при каждом вызове
   - Признак: высокий input token count
   - Решение: сократить SOUL.md, вынести в скиллы

4. **Большой memory/** — контекст раздувается
   - Признак: растущий расход при каждом вызове
   - Решение: compaction mode safeguard, чистка старых записей

5. **Неэффективная маршрутизация** — Кай делегирует не тому агенту
   - Признак: агент получает запрос не по теме, тратит токены на "не моя тема"
   - Решение: улучшить триггеры в AGENTS.md

## Формат отчёта

```
## Токены — YYYY-MM-DD HH:MM

Расход за сутки: $X.XX
Уровень: нормальный/повышенный/алерт/критический

По агентам:
- kai: ~$X.XX (N вызовов)
- technologist: ~$X.XX (N вызовов)
- titan: ~$X.XX (N вызовов)
- mentor: ~$X.XX (N вызовов)
- architect: ~$X.XX (N вызовов)

Аномалии: [есть/нет]
[Если есть — описание и рекомендация]
```

---

## Правила

1. Проверяй токены каждые 3 часа (каждый 3-й heartbeat)
2. При алерте ($5+) → немедленно сообщи Максиму с анализом причины
3. Предлагай конкретные меры экономии (какой агент, что изменить)
4. Логируй каждую проверку в memory/ даже если всё в норме
