# Скилл: Управление бэкапами

## Когда использовать
Запросы о бэкапах, резервном копировании, восстановлении данных.

---

## Ежедневный бэкап

```bash
# Создать бэкап
tar czf ~/.openclaw-backup-$(date +%F).tar.gz ~/.openclaw/

# Проверить что создался
ls -lh ~/.openclaw-backup-$(date +%F).tar.gz
```

Запускать: раз в 24 часа (каждый 24-й heartbeat).

---

## Ротация бэкапов

```bash
# Удалить бэкапы старше 7 дней
find ~/ -maxdepth 1 -name ".openclaw-backup-*.tar.gz" -mtime +7 -delete

# Проверить оставшиеся
ls -lh ~/.openclaw-backup-*.tar.gz
```

Хранить: последние 7 бэкапов (7 дней).

---

## Проверка целостности

```bash
# Проверить что архив не повреждён
tar tzf ~/.openclaw-backup-$(date +%F).tar.gz | head -20

# Размер архива (должен быть > 100KB)
ls -lh ~/.openclaw-backup-$(date +%F).tar.gz
```

Если архив < 100KB или повреждён → [АЛЕРТ] бэкап не прошёл.

---

## Восстановление

### Полное восстановление
```bash
# 1. Остановить gateway
openclaw gateway stop

# 2. Бэкап текущего (на всякий случай)
mv ~/.openclaw ~/.openclaw.broken.$(date +%F)

# 3. Восстановить из бэкапа
tar xzf ~/.openclaw-backup-YYYY-MM-DD.tar.gz -C /

# 4. Перезапустить
openclaw gateway restart

# 5. Проверить
openclaw status
openclaw doctor
```

### Восстановление одного файла
```bash
# Посмотреть содержимое бэкапа
tar tzf ~/.openclaw-backup-YYYY-MM-DD.tar.gz | grep "имя_файла"

# Извлечь конкретный файл
tar xzf ~/.openclaw-backup-YYYY-MM-DD.tar.gz -C /tmp/ путь/к/файлу

# Скопировать на место
cp /tmp/путь/к/файлу ~/.openclaw/путь/к/файлу
```

## Формат отчёта

```
## Бэкап — YYYY-MM-DD HH:MM

Статус: создан / ошибка
Файл: ~/.openclaw-backup-YYYY-MM-DD.tar.gz
Размер: XX MB
Бэкапов на сервере: N (из 7 допустимых)
Удалено старых: N

Проблемы: нет / [описание]
```

---

## Правила

1. Бэкап раз в сутки — НЕ пропускать
2. После бэкапа → проверить целостность (tar tzf)
3. После ротации → убедиться что осталось минимум 3 бэкапа
4. Восстановление → ТОЛЬКО с подтверждения Максима (кроме аварии)
5. Логировать каждый бэкап в memory/
