# System Monitor — Мониторинг VPS

## Команды проверки

### Диск
```bash
df -h /
```
Парсить: колонку "Use%" → если >80% → алерт

### RAM
```bash
free -m
```
Парсить: строку "Mem:" → used/total → если >90% → алерт

### CPU
```bash
top -bn1 | head -5
```
Парсить: "%Cpu(s)" → 100 - idle → если >95% → алерт

### OpenClaw Gateway
```bash
systemctl status openclaw
```
Парсить: "Active:" → если не "active (running)" → алерт

### Uptime
```bash
uptime
```
Парсить: "up X days" → если <1 дня → инфо (был перезапуск)

## Пороги алертов

| Метрика | Порог | Уровень |
|---------|-------|---------|
| Диск | >80% | АЛЕРТ |
| Диск | >90% | КРИТИЧЕСКИЙ |
| RAM | >90% | АЛЕРТ |
| RAM | >95% | КРИТИЧЕСКИЙ |
| CPU | >95% (устойчиво) | АЛЕРТ |
| Gateway | не работает | КРИТИЧЕСКИЙ |
| Uptime | <1 день | ИНФО |

---

## Формат алерта

```
[АЛЕРТ] Тип: описание
Значение: X% (порог: Y%)
Рекомендация: что сделать
```

### Пример
```
[АЛЕРТ] Диск: использование превышает порог
Значение: 84% (порог: 80%)
Рекомендация: удалить старые логи или увеличить диск
```

---

## Формат HEARTBEAT_OK

Если ВСЕ метрики в норме:
```
HEARTBEAT_OK
```

НЕ пиши Максиму. Молча верни HEARTBEAT_OK.

## Рекомендации по проблемам

### Диск полный
1. `du -sh /var/log/*` — проверить логи
2. `journalctl --vacuum-size=100M` — очистить systemd логи
3. `npm cache clean --force` — очистить кэш npm
4. Уведомить Максима для дальнейших действий

### RAM заполнена
1. `ps aux --sort=-%mem | head -10` — топ процессов по RAM
2. Обычно: OpenClaw gateway + Node.js
3. Уведомить Максима (возможно нужен рестарт)

### CPU загружен
1. `ps aux --sort=-%cpu | head -10` — топ процессов по CPU
2. Проверить не зациклился ли процесс
3. Уведомить Максима

### Gateway не работает
1. `journalctl -u openclaw -n 20` — последние логи
2. Уведомить Максима с текстом ошибки
3. НЕ перезапускать самостоятельно
