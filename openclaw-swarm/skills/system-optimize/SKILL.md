# Скилл: Оптимизация системы

## Когда использовать
Запросы на оптимизацию, тюнинг, улучшение производительности, ускорение, уменьшение расхода токенов.

---

## Чеклист оптимизации

### 1. SOUL.md всех агентов
```bash
wc -w ~/.openclaw/workspace/SOUL.md
wc -w ~/.openclaw/agents/*/SOUL.md
```
- **Норма:** < 1200 слов
- **Проблема:** > 1200 слов → модель игнорирует правила в конце
- **Решение:** Сократить, вынести детали в скиллы

### 2. Heartbeat интервалы
| Агент | Текущий | Оптимально |
|-------|---------|------------|
| kai | 60m | 60m (координация) |
| technologist | отключен | отключен (по запросу) |
| titan | 60m | 60m (напоминания) |
| mentor | 60m | 60m (дедлайны) |
| architect | 60m | 60m (мониторинг) |

- Если агент не делает ничего полезного в heartbeat → увеличить интервал или отключить
- Каждый heartbeat = токены. Пустой heartbeat = потерянные деньги.

### 3. Скиллы — качество
Для каждого скилла проверь:
- [ ] Есть YAML frontmatter (description, triggers)
- [ ] Размер < 500 слов (иначе сократить)
- [ ] Содержит ФАКТЫ, а не инструкции поведения
- [ ] Нет дублирования с SOUL.md
- [ ] Используется хотя бы 1 раз в неделю (иначе — кандидат на удаление)

### 4. Расход токенов по агентам
```bash
openclaw status
```
Анализ:
- Какой агент тратит больше всех?
- Есть ли heartbeat без полезного действия?
- Есть ли зацикленные ответы (loop detection)?

### 5. Размер memory/
```bash
du -sh ~/.openclaw/workspace/memory/
du -sh ~/.openclaw/agents/*/memory/
```
- > 10MB → почистить старые записи (> 30 дней)
- Compaction mode = safeguard (проверить в openclaw.json)

---

## Правила

1. Мелкие оптимизации (опечатки, форматирование, сокращение текста) → делай сам, логируй
2. Изменение heartbeat интервалов → делай сам, логируй, наблюдай 24ч
3. Удаление скиллов / изменение архитектуры → спроси Максима
4. Каждое изменение → запись в memory/ с датой
